A demo that behaves like the real thing

It isn't a set of screenshots. It runs two ways from the same rules, and every action you take is reversible.

Two engines, one contract

The same workflow runs as a pure in-browser demo and against an optional backend. Both follow one set of rules, checked on every change.

  • Static mode needs no server; open it and it works.
  • App mode persists to a backend with the identical behavior.
  • Gate scripts pin the two engines to the same contract.

Receipts you can undo

Finish work, clear a blocker, add a note: each action shows a receipt of exactly what changed, right where you clicked.

  • Every change is stated plainly, not silently applied.
  • One click, or ⌘Z, puts the work back as it was.
  • Undo covers actions, edits, notes, and new work alike.

Built to be checked

The interesting part of a portfolio demo isn't that it looks finished. It's whether it holds up, and this one is wired to prove it does.

Every button click-tested

CI drives a real browser that clicks every enabled control on every screen and fails the build if one does nothing.

Two engines pinned together

Gate scripts assert the static demo and the backend produce the same workflow, on every push.

Byte-budgeted & sealed

Public files ship under size budgets, and the build fails if anything internal or oversized leaks out.

What this demo doesn't claim: real integrations, authentication, or production scale. It's sample data with no login, by design: a portfolio piece, not a product.
